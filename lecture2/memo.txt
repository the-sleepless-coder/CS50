## what process does compiling go through 
preprocessing -> compiling -> assembling -> linking 

-preprocess : copy pastes the code that's used in the code that you're compiling
preprocessor directive (starts with #)
#include <cs50.h>, #include<stdio.h>

-compile : translated from C -> Assembly code
the CPU runs instructions of assembly code that are changed from source code

-assembling : change assembly code -> machine code

-linking : linking the files from preprocessing directive and main 
hello.c/cs50.c/stdio.c
thus linking all the preprocessed / main files that are compiled and assembled to a single program

#Chars/Strings
char : single characters represented by numbers
string : sequence of characters
Or basically it can be called an array of characters
that are saved backed to back in a form of an array

To know where the string actually ends, 
it has an additional byte that is filled with 0 to indicate the end of the array
or as to say the end of the whole series of continguous elements
(also known as a sentinel value that shows where the array ends: \0)
which is called nul ( with one l )

all 0 bits to show where the array ends

when you declare a type as string, 
it adds the nul byte to show where the given data ends

when you add double quotes that where the additional nul byte is added, 
to show where the sequence of characters end.

characters are distinguished by a single quote,
and thereby doesn't need a nul to show where it ends

printf does exactly that job of printing the characters till the end of the stream of characters.
