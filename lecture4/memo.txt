#Scope
because of the scope of a function, 
something that has happened within a function may not affect the variables in main

when you pass variables in a function you are passing by value
copies of the values are passed in(copy of x and y)

but no affect on the original version of x, y

int main(void){
    int x = 1;
    int y = 2;

    printf("x is %i, y is %i\n", x, y);
    swap(x, y);
    printf("x is %i, y is %i\n", x, y);
    
}

void swap(int a, int b)
{
    // tmp에 a를 담아두고, 
    // b에 tmp를 넣어둔다.
    int tmp = a;
    a = b;
    b=tmp;

}

# passing by reference(address)
pass by value is logically correct, but you need to pass the address so that you can actually change the value in the given address



pass by value를 할 수 밖에 없는 것은, 
함수마다 자기만의 지역 변수 공간(stack)을 가지고 있기 때문에 복사한 값을 넘길 수 밖에 없어서 그렇다.
그래서 애초에 저장된 주소 공간 자체가 다르기 때문에 값만 복사하고, 
swap function에서 실행된 코드는 main function에 반영되지 않는다.

primitive 변수라면 값 자체를 복사하고(동일한 값을 각자의 stack에 서로 다른 주소)
Object인 객체라면 heap객체의 주소를 복사한다.(동일한 주소 값의 heap에서 같은 주소)

그래서 java에서 primitive값을 main에서 호출해서 쓰면 pass by value를 하기 때문에 main에서 호출한 함수의 값이 변하지 않는 것이고, 
객체는 heap에 있는 주소를 복사하기 때문에 동일한 주소값의 변수를 수정하기 때문에 바뀌는 것이다.

#pass by reference
int main(void){
    int x = 1;
    int y = 2;

    // you need to pass on the address so that you can actually change the value in the given address
    printf("x is %i, y is %i\n", x, y);
    swap_ref(&x, &y);
    printf("x is %i, y is %i\n", x, y);
    
}

void swap_ref(int *a, int *b){
    int tmp = *a;
    *a = *b;
    *b = tmp;
}
