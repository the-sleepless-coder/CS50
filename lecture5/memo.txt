# Hash Table (specific way to implement a dictionary)
Hash function 
allows you do decide where some values go into memory

bucketize the value in a certain location to find it quickly
take a large domain and map it to a smaller range of values


large domain (input) -> hash function -> finite range of value 

combination of an array and linked list together

using the bucket as key
and in the key you put in the value as a linked list
to prevent the same buckets having the same value, you need a linked list in order to prevent that from happening

In the worst case scenario, 
time complexity could lead to O(N) 

So essentially N pointers to the start of a linked list,
is what you could say a hash table is

To reduce the probability of collisions,
you can increase the number of buckets in order to do so
(ex. look at the first 3 letters of the name)

But the downside would be that it would take more space
(some memory spaces would not be used and hence lead to the waste of it)

The average time complexity would be 
O(N/K)
K = number of buckets

Considering the amount fo names you would put and the size of the bucket K
you woule be able to design a hash function that has a time complexity close to O(1)

if you design the buckets so that there could be an average of more than 1 linked list,
the time complexity of the hash table would increase(bigger than 1)
